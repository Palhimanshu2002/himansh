Graphical User Interface

The most widely used type of Ul in use today is the Graphical User Interface (GUI). It accepts input through the key and mouse and displays the output on the user's computer screen. The actions in a GUI are usually performed through direct manipulation of the graphical elements.

The most common elements of GUI include:

  • Window

    It is an area on the screen that displays information. Contents in a window are displayed in the form of icons or lists. According to the size of the screen, windows can be minimized, resized, or maximized. They can be moved anywhere on the screen. A window may have one or more windows as its children. These child windows are drawn within the client area of the parent window.

  • Icon

    It is a small picture that represents a software application or a hardware device. An application window opens on clicking or double-clicking these icons.

  • Menu

    It is a list of choices presented to the user. A menu is placed at a visible (generally top) inside the application.

  • Pointer/Cursor

    It is an onscreen symbol that represents movement of a device that the user controls to select windows, icons, and menus.

    Definition: A VUI allows users to interact with a system using voice or speech commands. Virtual assistants like Siri, Google Assistant, and Alexa are examples of VUI.

Principles Attributes Of User Interface Design

UI DESIGN PRINCIPLES

  • Structure
  • Simplicity
  • Visibility
  • Feedback
  • Tolerance

Structure Principle

This principle is concerned with overall UI architecture.

The design of the inerface should be visually, theoretically, and linguistically clear.

Tips/Tecniques to support the Structure Principle

You should group the logically connected item to communicate and separate unrelated item to achieve visual organization.

  • Design page-specific navigation and access to information.
  • Keep the screen less cluttered and easier to understand.
  • Present the flow of action,information,rsponses, and visual preparations in a sensible order that is easy to remember and place in context.
    •

    Simplicity Principle

  • The design Should be simple to learn and use.
  • It should include only the elements that are most important for communication.
  • It should also make common task easy to perform
  • It should provide good shortcuts that are meaningfully related to longer procedures.
    •

    Tips/Tecniques to support the Simplicity Principle

    Ensure that you create natural design, which can be easily understood by the user based on their past experiences.
